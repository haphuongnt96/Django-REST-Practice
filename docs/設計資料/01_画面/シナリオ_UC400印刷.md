# ユースケース: 印刷

# 前提条件

- ユーザーはシステムにログイン済みであること。
- 申請一覧で検索された申請内容を選択し、申請詳細に遷移していること。

# 基本フロー

### 条件

### フロー

1. システムは申請詳細を表示する。
1. 申請者・承認者はシステムに印刷を指示する。
1. システムは申請の内容をPDFとして出力する。
    - 出力内容 : 申請情報、申請項目
    - 承認ルートは出力対象外

# 代替フロー

### 代替フロー 2-1 承認済みではない申請の場合

1. 該当申請の申請ステータスが承認済ではない場合
1. 申請者・承認者はシステムに印刷を指示できない。
